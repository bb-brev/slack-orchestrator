All Available Slack Surfaces

Surfaces are places where the Brev Operator can express itself. Each surface has different capabilities, constraints, and ideal use cases for the Brev portal.

Primary

Messages

Rich text messages in channels, DMs, and threads. Supports Block Kit layouts (up to 50 blocks per message), interactive components, file attachments, and unfurls.

Brev use: Follow-ups, check-in drafts, risk alerts, weekly digests, approval prompts, question routing

Key methods: chat.postMessage, chat.update, chat.postEphemeral, chat.scheduleMessage

Limits: 50 blocks/msg, 40K chars text, 1 msg/sec/channel (Tier 3)

Dashboard

App Home (Home Tab)

A persistent, per-user view that lives under the Brev bot in the sidebar. Up to 100 blocks. Always accessible, retains state until the app updates it. Fully customizable per user.

Brev use: Personal dashboard — my goals, my check-ins due, my action items, pending approvals, Operator activity log

Key method: views.publish (publishes/updates the Home tab for a specific user)

Limits: 100 blocks, no input blocks (use modals for forms)

Forms

Modals

Pop-up dialogs that capture focus. Support input fields, select menus, date pickers, radio buttons. Can chain up to 3 views in a view stack for multi-step wizards.

Brev use: Create/edit goals, submit check-ins, configure Operator settings, approve/reject actions with notes, multi-step wizards

Key methods: views.open, views.update, views.push

Trigger: Requires a trigger_id — can only open in response to user interaction (button click, slash command, shortcut)

AI-Native

Split View (AI Pane)

Side-by-side AI chat experience. User talks to the Operator in a pane while viewing channels/threads on the other side. Includes suggested prompts, loading states, and text streaming.

Brev use: Natural language queries ("what's at risk this week?"), conversational goal creation, deep-dive analysis, ad-hoc reports

Key methods: assistant.threads.setStatus, assistant.threads.setSuggestedPrompts, chat.startStream/appendStream/stopStream

Requires: Agents & AI Apps toggle + assistant:write scope. Paid Slack plan.

Documents

Canvases

Built-in documents in Slack. Can be attached to channels or standalone. Support markdown, links, videos, and workflow triggers. Persistent and collaborative.

Brev use: Meeting agendas, weekly reports, quarterly reviews, onboarding guides, team dashboards — long-form artifacts that persist

Note: Different from Block Kit — uses markdown formatting. Good for generated reports that need to live beyond a message.

Quick Access

Slash Commands

User types /brev + subcommand in the message composer. Instant access to any Brev action from any channel.

Brev use: /brev goals, /brev checkin, /brev risk, /brev assign @user goal-name, /brev report

Response: Can respond with ephemeral message (only sender sees), channel message, or trigger a modal

Block Kit Components for Brev

Block Kit is the UI framework for Slack messages, modals, and Home tabs. These are the components Brev Operator uses to create rich, interactive portal experiences.

Layout Blocks

sectionText + optional accessory (button, image, select). Goal cards, KPI readouts, check-in summaries.
headerLarge bold text. Report titles, section headers.
dividerVisual separator. Between goals in a report.
contextSmall text + images. Timestamps, owner avatars, data sources.
actionsGroup of interactive elements. Approve/reject buttons, status selectors.
inputForm field (modals only). Check-in text, KPI value, goal title.
rich_textRich formatted text. Formatted reports, meeting notes.

Interactive Elements

buttonClick action. Styles: primary (green), danger (red), default. Approve, reject, view in Brev, edit.
selectDropdown. Static, external, user, conversation, channel selects. Assign owner, pick team, set status.
multi_selectMulti-pick dropdown. Tag goals, add collaborators.
datepickerCalendar date selection. Set deadlines, schedule follow-ups.
overflow⋯ menu. More actions: reassign, archive, mark complete.
radioRadio button group. Status selection (on track/behind/at risk).
checkboxCheckbox group. Select action items to mark complete.

App Home — Personal Brev Dashboard

The App Home is a private, per-user space that lives under the Brev bot in the Slack sidebar. It has three tabs:

Home Tab ★

Fully customizable, persistent view. Up to 100 Block Kit blocks. Updated via views.publish.

Brev portal: Personal dashboard showing your goals, pending check-ins, action items, Operator activity log, and settings.

Messages Tab

DM conversation between the user and the bot. Standard messaging features.

Brev portal: Chat with the Operator. Ask questions, give instructions, provide context for follow-ups.

About Tab

Descriptive info about the app. Set during app configuration.

Brev portal: Help docs, configuration links, "Open in Brev" button.

Home Tab Layout for Brev Operator

Section 1: Status Summary

Your goals: 3 On Track, 1 Behind, 1 At Risk · 2 check-ins due · 4 pending action items

Section 2: Pending Actions

Check-in drafts awaiting approval · Action items from meetings · Operator recommendations to review

Section 3: My Goals

Goal cards with progress bars, status badges, owner, and quick-action buttons (Update, Check-in, View in Brev)

Section 4: Operator Activity

Recent Operator actions: "Sent follow-up to @jake re: API migration" · "Drafted Q2 pipeline check-in" · "Escalated QA capacity to @sarah"

Section 5: Settings

Button → opens modal: notification preferences, follow-up frequency, autonomy level, mute controls

Modals — Rich Forms & Multi-Step Wizards

Modals are pop-up windows that capture focus. They're the best surface for structured data input — creating goals, editing check-ins, configuring the Operator. Key features:

Capabilities

  • • Up to 3 views in a view stack (multi-step wizards)
  • • Input blocks: text, selects, date pickers, checkboxes, radio buttons
  • • Can push, update, or clear views dynamically
  • • Submit + cancel buttons with custom labels
  • • App can validate and return errors inline

Brev Use Cases

  • • Create Goal: Title → Owner → Key Results → Parent Goal (3-step wizard)
  • • Edit Check-in: Pre-filled draft, edit text, change status, submit
  • • Approve with Notes: View action details, add notes, approve/reject
  • • Operator Settings: Notification prefs, follow-up frequency, DM vs. channel
  • • Reassign Goal: Pick new owner from user select, add reason

Constraint: Modals require a trigger_id — they can only be opened in response to a user action (button click, slash command, shortcut). The Operator cannot proactively open a modal. This is by design — modals are always user-initiated.

Slack AI Features — Split View & Agent Experience

Requires Paid Slack Plan

Slack's Agents & AI Apps framework gives Brev Operator a native AI experience: side-by-side chat, suggested prompts, loading states, text streaming, and thinking-step visualization.

Split View (Side-by-Side Pane)

The Operator lives in the Slack top bar. Clicking it opens a side pane — user can chat with the Operator while viewing channels/threads on the other side. Conversations are auto-threaded.

Brev portal: "What's at risk this week?" → Operator streams a real-time analysis with Block Kit formatting. User never leaves their channel view.

Suggested Prompts

Present clickable prompt suggestions when a user opens a new conversation with the Operator. Contextual and personalized.

"prompts": [
  { "title": "📊 Weekly Report", "message": "Generate this week's exec summary" },
  { "title": "🎯 My Goals", "message": "Show all my goals and their status" },
  { "title": "⚠️ What's at Risk?", "message": "What goals are at risk right now?" },
  { "title": "✅ Pending Check-ins", "message": "What check-ins do I need to review?" }
]

Text Streaming

Stream long-form responses token-by-token using chat.startStream → chat.appendStream → chat.stopStream. Block Kit blocks can be added in stopStream for structured formatting after streaming completes.

Brev portal: Long analysis requests get streamed output so users see progress immediately. Final message includes formatted Block Kit cards.

Agent Thinking Steps

Display the Operator's reasoning process using task_card blocks and plan blocks. Two display modes:

Timeline mode:

Tasks appear as individual cards in sequence. Good for step-by-step operations.

Plan mode:

Tasks grouped in a comprehensive plan view. Good for multi-step analysis.

Brev portal: "Reading your 5 goals... Checking HubSpot pipeline... Analyzing meeting transcripts... Drafting risk assessment..." — visible thinking builds trust.

Events API & Real-Time Listening

The Events API pushes events to Brev Operator whenever specific things happen in Slack. Two delivery options: HTTP endpoints or Socket Mode (WebSocket).

Events Brev Operator Needs

Event Trigger Brev Use
app_mentionSomeone @mentions @brevPrimary interaction entry point — parse intent, route to action
message.imDM to the botPrivate conversations — personal check-ins, questions, configuration
message.channelsMessage in a channel the bot is inContext listening — detect action items, commitments, decisions (opt-in per channel)
reaction_addedUser adds a reaction to a message✅ reaction on Operator draft → approve and execute. 👀 → acknowledge. ❌ → dismiss.
app_home_openedUser opens the Brev App HomeRefresh dashboard with latest data — goals, pending items, activity log
member_joined_channelBot added to a channelIntroduce Brev Operator, offer to connect channel to a team/goal
assistant_thread_startedUser opens the AI split paneSet suggested prompts, initialize context for conversational queries
assistant_thread_context_changedUser switches context in the split paneUpdate Operator context to reflect current channel/thread being viewed

Socket Mode vs. HTTP

Socket Mode (WebSocket)

No public URL needed. Real-time. Good for development and single-instance deployments. Not ideal for horizontal scaling.

HTTP Request URLs ★

Production-grade. Scalable behind load balancers. Standard webhook pattern. Recommended for Brev's multi-tenant architecture.

Rate Limits & Constraints

Method Tier Rate Impact on Brev
chat.postMessageSpecial1 msg/sec/channel, 1 msg/sec/user for DMsQueue outbound messages. Batch digests. Don't spam channels.
chat.updateTier 3~50/minFine for updating approval statuses. Don't poll-update.
views.openTier 4~100/minNo concern — user-triggered.
views.publishTier 4~100/minHome tab updates. Batch per-user refreshes.
conversations.historyTier 3~50/minCache channel history. Don't re-read every cycle.
users.infoTier 4~100/minCache user profiles at install. Refresh periodically.

Multi-Tenant Considerations

Rate limits are per-app-per-workspace. Each Brev customer workspace has its own rate limit budget. For a 100-person org, 1 msg/sec is 3,600 msgs/hour — more than enough. The constraint isn't rate limits; it's user tolerance for message volume.

Implementation Recommendation

Slack SDK: Bolt for JavaScript

Use Bolt for JS — Slack's official framework. Handles OAuth, event subscriptions, interactivity, Socket Mode, and all the plumbing. Matches Brev's existing TypeScript stack.

Phase 1: Core Portal

  • • Bolt for JS setup with HTTP event delivery
  • • OAuth install flow (multi-tenant)
  • • @mention handling → intent detection → CRUD routing
  • • Block Kit message templates for goals, check-ins, reports
  • • Home Tab with personal dashboard
  • • 3 slash commands: /brev goals, /brev checkin, /brev help

Phase 2: Rich Interactions

  • • Modal forms for goal creation, check-in editing
  • • Reaction-based approvals (✅ / ❌)
  • • AI split view with suggested prompts
  • • Text streaming for analysis responses
  • • Full slash command suite
  • • Canvas integration for long-form reports

Communication Rules

  • Orchestrator → Agents: Spawns agents with a task prompt + scoped context. Agent receives only the data it needs (principle of least privilege).
  • Agents → Orchestrator: Return structured results (success/failure/blocked + artifacts). If blocked, include a structured question.
  • Coordinator → Humans: All human-facing communication goes through the Coordinator. Other agents never post to Slack directly.
  • Executor → Brev API: All write operations go through the Executor. Other agents produce drafts/recommendations; the Executor applies them after approval.
  • No peer-to-peer: Agents don't talk to each other directly. The Orchestrator coordinates all inter-agent work, preventing loops and conflicts.

03 🧠 Orchestrator Agent Core · v1

Identity

The AI COO's brain

The only always-on agent. Runs the planning loop, evaluates org state, decides what needs to happen, and delegates to specialized agents.

Model & Cost

Planning loop: Claude Opus (complex reasoning, multi-step prioritization)

Reactive routing: Claude Sonnet (fast intent detection)

Est. cost: $50–150/org/mo depending on org size and loop frequency

Lifecycle

Always-on per org. Runs on a cron schedule (every 2–4 hours during business hours). Also invoked reactively on @mentions and events.

Session: Persistent context window per org, compacted periodically.

What It Does — Decision Tree

PLANNING LOOP (cron: every 2-4h during business hours)
│
├─ Scan Goals
│  ├─ At Risk / Behind → Spawn Drafter(risk_report) + Coordinator(escalate)
│  ├─ Check-in Overdue → Spawn Drafter(checkin_draft) + Coordinator(deliver_dm)
│  └─ On Track, No Update in 2wk → Spawn Coordinator(gentle_nudge)
│
├─ Scan Action Items
│  ├─ Past Due → Spawn Coordinator(follow_up) with meeting transcript context
│  ├─ Due Today → Spawn Coordinator(reminder) with low urgency
│  └─ Completed but Unverified → Spawn Integrator(verify_from_source)
│
├─ Scan Cadences
│  ├─ Meeting in <1h → Spawn Drafter(agenda)
│  ├─ Meeting ended <2h ago → Spawn Drafter(action_items_from_transcript)
│  └─ QBR this week → Spawn Drafter(quarterly_report)
│
├─ Scan Integrations
│  ├─ KPI threshold crossed → Spawn Coordinator(alert) + Drafter(context)
│  └─ New data available → Spawn Integrator(sync) + Executor(update_kpi)
│
└─ Prioritize & Throttle
   ├─ Max 5 outbound messages per user per day (prevent spam)
   ├─ Priority: critical blockers > overdue check-ins > scheduled reports > nudges
   └─ Respect quiet hours (configurable per org)

Example: A Single Planning Cycle

Tuesday 10:00 AM, 50-person SaaS company. The Orchestrator runs its planning loop:

  1. Scans 12 company goals: 2 At Risk (NPS down, pipeline behind), 1 Behind (hiring target), 9 On Track.
  2. Finds 8 overdue check-ins: 5 are 3+ days overdue (high priority), 3 are 1 day overdue (low priority).
  3. Detects upcoming QBR: Thursday at 2pm. No agenda prepared yet.
  4. Checks integration data: HubSpot shows pipeline dropped 12% this week. Linear shows sprint velocity down.
  5. Prioritizes actions: (1) Pipeline alert → VP Sales, (2) 5 overdue check-in drafts, (3) QBR agenda, (4) NPS trend analysis.
  6. Spawns agents: 1 Drafter for QBR agenda, 5 Drafters for check-ins, 1 Coordinator for pipeline alert, 1 Analyst for NPS deep-dive.
  7. Throttles output: VP Sales gets the pipeline alert now. Check-in drafts staggered over 2 hours to avoid flooding. QBR agenda scheduled for Wednesday morning.

04 📝 Drafter Agent Core · v1

Identity

The writer

Produces all written artifacts: check-in drafts, weekly reports, meeting agendas, risk analyses, executive summaries. Pulls data from Brev APIs and integrations, synthesizes into human-ready output.

Model & Cost

Default: Claude Sonnet (good writing quality, fast, cost-effective)

Exec reports: Opus for quarterly reviews and board-level summaries

Est. cost: $0.02–0.15 per draft depending on complexity

Lifecycle

Spawned per task. Receives a specific brief from the Orchestrator ("draft Q2 pipeline check-in for @mike using HubSpot data + forecast call notes"). Produces artifact. Returns to Orchestrator. Session deleted.

Artifact Types

Check-in Drafts

Input: Goal details, KPI data from integrations, recent meeting transcripts, previous check-ins.

Output: Status assessment, progress narrative, blockers identified, suggested next steps. Pre-formatted for Slack Block Kit message with approve/edit buttons.

Volume: 5–50 per week depending on org size and check-in cadence.

Weekly/Monthly Digests

Input: All goals + status changes, KPI trends, completed check-ins, action items resolved, risks detected.

Output: Executive summary with sections: progress vs plan, risks & slippage, wins & momentum, team health, action items. Formatted for Slack channel post or Canvas.

Volume: 1–4 per week per org.

Meeting Agendas

Input: Cadence definition, open action items from previous meeting, goal status changes since last review, suggested discussion topics from Orchestrator.

Output: Structured agenda with time allocations, data tables, decision points flagged, and "what to prepare" per attendee.

Volume: 2–10 per week depending on cadence count.

Risk Assessments

Input: Goal at risk, KPI trend data, meeting transcript excerpts mentioning blockers, historical pattern.

Output: Root cause analysis, impact assessment, 2–3 recommended actions with trade-offs, suggested decision-maker to route to.

Volume: 1–5 per week (triggered by Orchestrator detection).

Action Item Extraction

Input: Meeting transcript from Brev's cadence engine. Attendee list with roles.

Output: Structured list: {action, owner, deadline (inferred), confidence, source quote}. Ready for Executor to create in Brev.

Volume: Post-meeting, 2–10 per week.

Escalation Briefs

Input: Blocked goal/task, context from Brev data + meeting history, stakeholder map.

Output: Decision brief for escalation: situation summary, options (2–3), trade-offs for each, recommended action. Formatted for DM to decision-maker.

Volume: 1–3 per week.

05 📣 Coordinator Agent Core · v1

Identity

The human interface

The ONLY agent that talks to humans. Handles all Slack interactions: delivering drafts, collecting approvals, routing questions, managing follow-up conversations. Understands tone, timing, and social dynamics.

Model & Cost

Delivery/routing: Haiku (fast, cheap — just formatting and sending)

Conversations: Sonnet (when the human replies and needs a thoughtful response)

Est. cost: $0.005–0.05 per interaction

Lifecycle

Event-driven. Activated by: Orchestrator dispatch, Slack events (reaction, thread reply, @mention), approval timeout. Manages conversation state across multiple Slack interactions.

Interaction Patterns

Deliver & Collect

Takes a Drafter artifact, formats it as a Slack Block Kit message, delivers to the right surface (DM, channel, thread). Attaches action buttons (Approve, Edit, Dismiss). Listens for the response. Routes the response back to the Orchestrator.

Orchestrator → Coordinator: "deliver check-in draft to @mike via DM"
Coordinator → Slack: posts formatted message with buttons
Human clicks "Approve" → Coordinator → Orchestrator: "approved, execute"
Orchestrator → Executor: "submit check-in to Brev API"
Follow-Up Conversations

When a human replies to a follow-up with context ("the migration is delayed because we're waiting on the security review"), the Coordinator parses the response, extracts the key information (blocker = security review), updates the Orchestrator, and may ask a clarifying question ("Got it — should I flag this to @sarah who owns the security review, or are you handling it?").

Escalation Delivery

Takes an escalation brief from the Drafter and delivers it to the decision-maker with carefully framed options. Manages the conversation: "I've flagged the QA capacity issue to @sarah with three options. She chose option 2 (bring in a contractor). I'll update the goal timeline and notify the team."

Social Intelligence

Understands: don't ping the CEO about a low-priority check-in at 9pm. Don't follow up with someone who just submitted 3 check-ins (they're engaged, leave them alone). Don't escalate to a manager during their vacation. Reads user availability from Slack status + calendar data.

06 ⚡ Executor Agent Core · v1

Identity

The doer

The only agent that writes to Brev. Takes approved actions and executes them: submitting check-ins, updating goal status, creating action items, reassigning owners. Runs only after explicit human approval (or Orchestrator auto-approval for pre-approved action types).

Model & Cost

Default: Haiku (structured API calls, no creativity needed)

Est. cost: $0.002–0.01 per action (cheapest agent)

Lifecycle

Shortest-lived agent. Spawned with a specific action + parameters. Executes. Reports success/failure. Session deleted. Every execution is logged to the audit trail.

Every execution is logged: who approved it, what changed, previous value, new value, timestamp. This audit trail is displayed in the Brev web app and accessible to admins.

07 🔌 Integrator Agent Core · v1

Identity

The data bridge

Connects Brev to the outside world. Pulls data from HubSpot, Linear, Jira, GitHub, Google Sheets, Salesforce, and more. Reconciles external state with Brev's model. The reason the Operator can draft check-ins with real data.

Model & Cost

Default: Haiku (structured API calls, data mapping)

Est. cost: $0.005–0.02 per sync operation

Lifecycle

Dual mode: (1) Scheduled — runs on cron to sync integration data (every 1–4 hours). (2) On-demand — Orchestrator or Drafter requests fresh data for a specific task.

Supported Integrations (v1)

CRM / Revenue
  • HubSpot: Pipeline value, deal stages, forecast, activity metrics. Maps to ARR/pipeline KPIs.
  • Salesforce: Opportunity pipeline, win rates, rep activity. Maps to revenue goals.
Engineering
  • Linear: Sprint velocity, issue counts, cycle time. Maps to engineering goals.
  • GitHub: PR throughput, CI pass rate, deploy frequency. Maps to dev velocity goals.
  • Jira: Story points completed, sprint burndown, backlog size.
Analytics / Data
  • Google Sheets: Custom KPI tracking sheets (manual metrics not in other tools).
  • Looker / Metabase: Dashboard metric queries. Auto-track KPIs from BI tools.
Productivity
  • Google Calendar: Meeting schedules, cadence timing, attendee availability.
  • Google Docs: Shared documents referenced in goals.

How Integration Data Flows

Integrator → queries HubSpot API → gets pipeline: $1.2M
  → compares to Brev KPI "New Pipeline" current value: $1.4M
  → detects: pipeline dropped $200K since last sync
  → returns to Orchestrator: { kpi: "pipeline", delta: -200K, threshold_breach: true }
  → Orchestrator decides: spawn Drafter(alert) + Coordinator(notify_vp_sales)

08 🔎 Analyst Agent Advanced · v2

Identity

The detective

Goes beyond surface-level reporting. When the Orchestrator detects a problem ("NPS dropped 15 points"), the Analyst investigates why. Cross-references multiple data sources, identifies correlations, and produces evidence-based analysis.

Model & Cost

Default: Opus (complex reasoning, multi-source synthesis)

Est. cost: $0.50–2.00 per analysis (highest per-task cost)

When It's Needed

Triggered when: KPI shows unexpected trend, goal status changes to At Risk, user asks "why is X happening?", quarterly deep-dives, board prep.

Analysis Types

Root Cause Analysis

"Pipeline dropped 20% this month" → Analyst checks: deal stage conversions, rep activity, average deal size, time-in-stage, new deal creation rate. Finds: 3 large deals slipped from commit to negotiate, reducing pipeline by $800K. Rep activity is flat — not a volume problem, it's a conversion problem in late-stage deals.

Trend Detection

Runs weekly on all KPIs. Identifies: accelerating trends (good or bad), plateau patterns (stalled progress), seasonal patterns, anomalies. Surfaces trends that humans might not notice until the quarterly review.

Cross-Goal Correlation

"Engineering velocity down 15%" + "QA backlog up 40%" + "3 meetings flagged deployment blockers" → Analyst connects these signals: QA capacity is the bottleneck, it's affecting eng velocity and deployment goals. Single root cause, multiple symptoms.

What-If Scenarios

"If we push the V2 launch by 2 weeks, what goals are affected?" → Analyst traces dependencies: launch goal pushes, marketing campaign timing shifts, Q2 ARR forecast adjusts. Produces a decision brief with quantified impact.

10 🎯 OKR Coach Agent Advanced · v2

The mentor

80%+ of successful OKR companies have dedicated OKR coaches. This agent replaces that role. Reviews goal quality, suggests improvements, identifies alignment gaps, and teaches teams OKR best practices — all through Slack conversations.

Model: Opus (nuanced feedback on goal quality requires strong reasoning)

Knowledge: Trained on OKR methodology (Doerr, Wodtke, Niven & Lamorte), Brev best practices, org-specific patterns

Capabilities:

  • • Review goal quality: measurable? ambitious enough? aligned?
  • • Suggest key results for a given objective
  • • Identify cascading alignment gaps
  • • Post-cycle retrospective analysis
  • • Flag goals that are vanity metrics vs. real outcomes
  • • Suggest goal consolidation (too many goals = no focus)
Example Interaction

User: "@brev review my team's Q3 OKRs"

Coach: "Reviewed 4 objectives and 12 key results for the Product team. Issues found:

  • • KR 'Improve user experience' is not measurable. Suggest: 'Increase task completion rate from 67% to 85%' or 'Reduce time-to-value for new users from 12min to 5min.'
  • • Objective 'Launch V2' has 5 key results — most effective OKRs have 2–3. Consider splitting into two objectives or consolidating KRs.
  • • No alignment link between your 'Reduce churn' KR and the company goal 'Improve NRR to 130%.' Should this be a child goal?"

11 🚀 Onboarding Agent Enterprise · v3

Guides new users and teams through Brev setup via Slack DMs. Personalized onboarding flow based on role (exec vs. manager vs. IC), team, and org maturity. Reduces time-to-value from weeks to days.

Onboarding Flow
  • 1. Welcome DM + role detection
  • 2. Walk through creating first goal (via modal)
  • 3. Connect integrations relevant to their role
  • 4. Show first check-in draft, explain Operator
  • 5. Set personal preferences (notification freq, DM vs channel)
  • 6. Introduce to team's existing goals + upcoming cadences
Team Onboarding
  • • Guided OKR-setting workshop via Slack (Coach agent assists)
  • • Template goals for common team types (Sales, Eng, Product, CS)
  • • Integration setup wizard per team
  • • First cadence scheduling
  • • 30-day health check: "Your team has been on Brev for 30 days. Here's what's working and what to improve."

12 🛡️ Compliance Agent Enterprise · v3

Audits org-wide goal health, process adherence, and data quality on a scheduled basis. Enterprise customers need this for board reporting, investor updates, and regulatory compliance (SOC2 evidence of operational controls).

What It Audits
  • • Goal coverage: does every team have OKRs? Any orphaned goals?
  • • Check-in compliance: % of goals with recent updates
  • • Alignment integrity: are child goals actually aligned to parents?
  • • Stale data: KPIs not updated in 30+ days
  • • Operator action audit: all AI actions logged with approver
  • • Data quality: duplicate goals, missing owners, incomplete KRs
Output
  • • Monthly compliance report (PDF/Canvas)
  • • Health score per team (0–100)
  • • Automated fix suggestions with 1-click apply
  • • Exportable audit trail for SOC2 evidence
  • • Board-ready OKR summary generator

14 Agent Design Patterns

1. Principle of Least Privilege

Each agent gets only the tools it needs. Drafters can't write. Executors can't read beyond their task scope. The Coordinator can't access Brev data directly. This prevents cascading failures and makes the system auditable.

2. Single Responsibility

Each agent does one thing well. The Drafter doesn't decide what to write — the Orchestrator decides. The Coordinator doesn't decide what to say — the Drafter writes it. This makes agents replaceable, testable, and cost-optimizable (use the cheapest model that can do the job).

3. Human-in-the-Loop by Default

Every write operation goes through the Executor, which only runs after approval. Approval can be: explicit (button click), implicit (✅ reaction), or pre-approved (org admin configured auto-approval for this action type). Never silent writes.

4. Fail Gracefully, Explain Clearly

When an agent fails (API error, model refusal, timeout), it returns a structured error to the Orchestrator, which decides: retry, skip, or escalate to human. The Coordinator always explains what happened in plain English: "I tried to pull your pipeline data from HubSpot but the connection timed out. I'll try again in 30 minutes, or you can update manually."

5. Stateless Agents, Stateful Orchestrator

Worker agents (Drafter, Executor, Integrator, Coordinator) are stateless — they receive everything they need in the task prompt and return results. Only the Orchestrator maintains persistent state (task queue, completion history, user interaction patterns). This makes worker agents horizontally scalable and cheap to replace.

16 Which Agents Does a Company Need?

Seed-Stage Startup (10–20 people)

3 agents

CEO is the de facto COO. Goals are informal. Meetings are unstructured. The Operator reduces the CEO's ops overhead so they can focus on product and customers.

🧠 Orchestrator (lightweight) 📝 Drafter 📣 Coordinator

Key value: Weekly digest to the team. Check-in reminders that actually draft the update. Meeting action item tracking. No Chief of Staff needed yet.

Growth-Stage Company (50–200 people)

5 agents

Multiple teams with their own OKRs. Regular cadences (standups, QBRs). Integrations matter. The Operator replaces the Chief of Staff's operational coordination work.

🧠 Orchestrator 📝 Drafter 📣 Coordinator ⚡ Executor 🔌 Integrator

Key value: Auto-tracked KPIs from HubSpot/Linear/GitHub. Proactive follow-ups on meeting commitments. Risk detection before QBR. $150K+ in ops coordination savings.

Scale-Up / Enterprise (200–2000 people)

8+ agents

Complex org structure. Board reporting requirements. Multiple business units. Cross-functional dependencies. The Operator is the operational nervous system connecting all teams.

🧠 Orchestrator 📝 Drafter 📣 Coordinator ⚡ Executor 🔌 Integrator 🔎 Analyst 🎯 OKR Coach 🛡️ Compliance

Key value: Cross-BU dependency tracking. Board-ready reporting. Goal health auditing. OKR coaching at scale. Root cause analysis when metrics slip. Saves multiple ops headcount.
